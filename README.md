# JOKER — 뇌 모양 AI 비서 채팅

아이언맨 자비스 스타일의 3D 뇌 비주얼 + Claude API 연동 채팅 웹앱.

## 부서별 뇌 영역

대화 주제를 7개 부서로 분류해 뇌의 담당 영역이 부서 색으로 발광합니다.
분류는 Claude가 답변 앞에 붙이는 `[부서:팀명]` 태그(서버가 제거 후 컨트롤
헤더로 전달)로 이루어지며, 백엔드가 없을 땐 키워드 분류로 폴백합니다.

| 부서 | 뇌 영역 | 색 |
|---|---|---|
| 전략기획팀 | 전두엽(앞) | 퍼플 |
| 마케팅팀 | 두정엽(위) | 핑크 |
| 영업팀 | 우측두엽 | 그린 |
| 설계팀 | 좌측두엽 | 블루 |
| 시공팀 | 후두엽(뒤) | 오렌지 |
| 정산팀 | 소뇌 | 골드 |
| 법무팀 | 뇌간·하부 | 레드 |

부서 구성 변경은 `js/depts.js`(색·키워드)와 `api/chat.js`(팀명 매핑·시스템
프롬프트)를 함께 수정하면 됩니다.

## 컴퍼니 메모리 (웹에서 직접 업데이트)

헤더의 톱니바퀴 버튼 → COMPANY MEMORY 패널에서 회사 공통 정보와 부서별
정보를 코드 수정 없이 등록·수정할 수 있습니다. 내용은 브라우저
(localStorage)에 저장되고 매 대화마다 조커의 시스템 프롬프트에 자동
주입되어, 조커가 이미 알고 있는 비서처럼 응대합니다.

브라우저별 저장이므로 기기 간 공유가 필요해지면 Vercel KV 등 서버 저장소로
확장하면 됩니다 (`api/chat.js`의 `buildKnowledgeBlock` 지점).

## 스킬 (업무 절차 등록)

같은 설정 패널의 SKILLS 탭에서 특정 업무의 절차·양식·톤을 스킬로 등록할 수
있습니다. 스킬마다 이름·발동 키워드·지침을 적어두면, 사용자 메시지에 발동
키워드가 포함된 대화에서만 해당 스킬 지침이 조커의 시스템 프롬프트에
주입됩니다 (요청당 최대 3개, 매칭 점수순). 스킬이 발동하면 보낸 메시지 아래에
`⚡ SKILL · 이름` 배지가 표시됩니다.

컴퍼니 메모리가 "조커가 아는 것"(항상 주입)이라면, 스킬은 "조커가 일하는
방법"(필요할 때만 주입)입니다. 저장 위치는 메모리와 동일하게 브라우저
localStorage이며, 서버 주입 지점은 `api/chat.js`의 `buildSkillBlock`입니다.

## 구성

| 경로 | 역할 |
|---|---|
| `index.html` | 페이지 마크업 + 스타일 |
| `js/brain3d.js` | Three.js 파티클 뇌 (GLSL 셰이더 맥동/신호, GSAP 상태 전환, FPS 거버너) |
| `js/brain2d.js` | 2D 캔버스 폴백 뇌 (WebGL 미지원·저사양 기기) |
| `js/chat.js` | 채팅 UI, 스트리밍 타이핑 효과, 오프라인 데모 모드 |
| `js/settings.js` | 설정 패널 셸 + 컴퍼니 메모리 탭 |
| `js/skills.js` | 스킬 탭 (등록 UI + 키워드 매칭) |
| `js/main.js` | 3D/2D 부트 선택 및 폴백 스왑 |
| `api/chat.js` | Claude API 스트리밍 백엔드 (Vercel 서버리스 함수 호환) |
| `server.js` | 로컬 개발 서버 |
| `vendor/` | three.js, GSAP (오프라인 동작을 위해 번들) |

## 실행

### 로컬 (실제 LLM 대화)

```bash
npm install
ANTHROPIC_API_KEY=sk-ant-... npm start
# http://localhost:3000
```

API 키는 서버에서만 사용되며 프론트엔드에 노출되지 않습니다.

### Vercel 배포

저장소를 Vercel에 연결하고 환경 변수 `ANTHROPIC_API_KEY`만 설정하면
`api/chat.js`가 자동으로 서버리스 함수로 배포됩니다.

### 백엔드 없이 (데모 모드)

`index.html`을 브라우저로 그냥 열면 조커가 내장된 위트 응답으로 대답하는
오프라인 데모 모드로 동작합니다.

## 옵션

- 모델 변경: 환경 변수 `JOKER_MODEL` (기본 `claude-opus-4-8`)
- 강제 렌더 모드: URL에 `?mode=2d` 또는 `?mode=3d`
- FPS 거버너 끄기(테스트용): `?gov=0`

## 성능 폴백

3D 뇌는 60fps를 유지하지 못하면 단계적으로 품질을 낮춥니다:
해상도(DPR) 축소 → 파티클/연결선 수 축소 → 2D 캔버스 뇌로 전환.
모바일은 처음부터 파티클 수를 절반으로 시작합니다.
